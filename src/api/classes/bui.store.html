bui.store Class

Defined in: /Users/wilson/personal/htdocs/opensource/bui-rollup/src/scripts/core/bui.store.js:9
Module: Core

Available since 1.5.0

数据存储订阅器

在数据更改的时候,会触发数据更改事件. 公共数据在 index.js 只要初始化一次, 类似vuejs的用法,但又不完全一致.

预览地址: demo

方法说明:

get: 获取所有数据或者某个字段数据
set: 保存某个字段数据,并触发相关事件
del: 删除某个字段,并触发相关事件
on: 绑定事件,事件名为数据的字段名
off: 取消绑定事件,事件名及回调需要跟on保持一致
trigger: 手动触发事件变更

Parameters:

  • [option] Object optional
    • [scope] String optional

      [数据的范围唯一标识]

    • [isPublic] Boolean optional

      [是否作为公共数据, 默认: false ( 在模块里面使用) | true (在index.js 路由初始化以后使用)]

    • [data] Object optional

      [动态监听的对象,当变更会自动触发相关联的选择器,需要提前定义]

    • [computed] Object optional

      [需要动态计算改变的相关联的时候,例如 fullName ,依赖于 data.firstName,data.lastName]

    • [watch] Object optional

      [监听的数据字段,在数据改变的时候,做对应的其它事情]

    • [method] Object optional

      [定义的方法,便于拿取相关数据]

Example:

var store = bui.store({ scope: "app", data: { page: 1 } });

// 这样就会设置内容为1

你还可以监听该对象改变的时候,做些其它事情 // 监听 page 改变的时候触发 store.on("page",function(2){

})

// 设置以后就会触发 store.page = 2;

方法2:

// 监听 page 下 name 改变的时候触发 store.on("page.name",function(2){

})

// 修改 page 数据 store.set("page.name",1321);

// page.name = 1321;

取消监听某个数据

Example:

  var log = function(val){
     console.log(val)  // 2
   }
   // 监听 page 改变的时候触发
   store.on("page",log)

   // 取消
   store.off("page",log)

   // 修改 page 数据
   store.set("page",2);

监听数据的修改, on事件监听必须在 set 数据修改之前.

  • [callback] Function optional

  // 监听 page 改变的时候触发

   })

     console.log(val)  // 3
   })

   // 修改 page 数据
   store.set("page",2);

   // 修改 page.name 数据
   store.set("page.name",3);
